HTML

  • The HEAD element contains information about the webpage
    • Defines three elements:
      • The charset element -> sets character encoding
      • The http-equive element sets an HTTP header for content
      • The viewport element sets the default view for our webpage
    • The HREF attribute points to the URL for the link for both BODY and HEAD
    • Can embed an entire stylesheet directly in the HTML file using a STYLE element in the HEAD element
      • The rules only apply to that page
        • can configure CSS classes and IDs -> used to identify elements, to style multiple elements in the page
    • a LINK element creates the connection to the external file. Use:
      • REL attribute specifies the relationship between the current document and the linked document resource
      • HREF links to external resource, using a relative path
  • The BODY element represents the visible content shown to the user
    • When working with visible HTML elements, the content is contained between an opening tag and a closing tag
    • DIV creates a division in the page
    • SPAN creates an inline division
    • The H1, H2, H3, H4, H5, and H6 elements represent the level of heading a given text block represents
      • H1 should only be used once per page + headers should be used in ascending order
    • The P element represents a paragraph or block of text
    • The UL, OL, and LI elements represent unordered lists, ordered lists, and list items
    • The IMG element contains information about images that are displayed on the webpage
      • The SRC attribute is the most important because it defines the location of the image
      • The ALT attribute contains a text string that describes the appearance and functionality of the image.
        • Helps with screen readers, assistive devices, and can improves SEO rankings
    • The BR element creates a line of empty space, or a line break
    • The A element (which stands for anchor) creates links to the same webpage or other webpages
  • The FOOTER appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
  • The SECTION elements are semantic elements that define a section in a document that contains thematic content
    • Creates a class =“___”; can be referenced in CSS
  • To STYLE an element directly in the HTML file use inline CSS
    • Inline styles apply to a specific HTML tag, with a style attribute from a CSS rule
    • + Quick and permanent changes
    • - less flexible

JavaScript

  • JavaScript (JS), a high-level dynamic programming language, allows users to interact with websites
  • One of the most popular modern web technologies
  • Can power websites with effects, interactivity, data communication, and more!
  • A weakly typed language allowing implicit type conversion when an operation involves mismatched types instead of throwing type errors.
    • Click a button, retrieve data, or fill out a form
  • To do this JavaScript defines a handful of coercion rules.
    • The primitive coercion process is used where a primitive value is expected, but there's no strong preference for what the actual type should be.
  • Developers have written a variety of tools on top of the core JavaScript language, unlocking a vast amount of functionality with minimum effort
    • Browser Application Programming Interfaces (APIs) built into web browsers, providing functionality such as dynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a user's webcam, or generating 3D graphics and audio samples.
    • Third-party APIs that allow developers to incorporate functionality in sites from other content providers, such as Twitter or Facebook.
    • Third-party frameworks and libraries that you can apply to HTML to accelerate the work of building sites and applications.
  • JS RULES
    • MUST link in the html sheet under footer before body closing tabs
      • the script.js file is linked within the BODY element.
      • ex: SCRIPT src="./assets/script.js" SCRIPT
    • A semicolon at the end of a line indicates where a statement ends
    • Case sensitive
    • All types except OBJECT define immutable values represented directly at the lowest level of the language aka PRIMITIVE VALUES
    • All PRIMITIVE types, except NULL, can be tested by the TYPEOF operator.
    • STRINGS are immutable. This means that once a string is created, it is not possible to modify it
    • OBJECTS are the only mutable values
    • FUNCTIONS are also OBJECTS with the additional capability of being CALLABLE.
    • When representing dates, the best choice is to use the built-in Date utility in JavaScript.
  • NULL vs UNDEFINED
    • UNDEFINED is the absence of a value; NULL is the absence of an object
    • NULL is a keyword; UNDEFINED is a normal identifier that is also a global property.
  • NUMBER vs BIGINT
    • NUMBER
      • A double-precision 64-bit binary format IEEE 754 value.
      • Can store positive floating-point numbers between 2^-1074 (Number.MIN_VALUE) and 2^1024 (Number.MAX_VALUE)
      • And negative floating-point numbers between -2^-1074 and -2^1024
      • Can only safely store integers in the range -(253 - 1) (Number.MIN_SAFE_INTEGER) to 253 - 1 (Number.MAX_SAFE_INTEGER).
        • Outside this range, JavaScript can no longer safely represent integers
        • Unsafe intergers will instead be represented by a double-precision floating point approximation.
        • Check if a number is within the safety range using Number.isSafeInteger().
    • BIGINT
      • A numeric primitive that can represent integers with arbitrary magnitude
      • Can safely store and operate on large integers even beyond the safe integer limit (Number.MAX_SAFE_INTEGER) for Numbers
      • Created by appending N to the end of an integer or by calling the BigInt() function.
      • Can use most operators with BigInts
  • JS SYNTAX
    • VAR ____ declares a variable
    • STRINGS need ""
    • CONSOLE.LOG outputs a message to the web console by adding an argument
    • NaN ("Not a Number") is a special kind of number value that's typically encountered when the result of an arithmetic operation cannot be expressed as a number.
      • It is also the only value in JavaScript that is not equal to itself.
    • ACCESSOR pairs a key with one of two ACCESSOR functions (get and set) to retrieve or store a value.
    • GET a function called with an empty argument list to retrieve the property value whenever a get access to the value is performed
    • SET a function called with an argument that contains the assigned value. Executed whenever a specified property is attempted to be changed.
    • ENUMERABLE a boolean value indicating if the property can be enumerated by a for...in loop.
    • CONFIGURABLE a boolean value indicating if the property can be deleted, can be changed to a data property, and can have its attributes changed.
    • OBJECTS are ad-hoc key-value pairs, so they are often used as MAPS.
      • Can be ergonomics, security, and performance issues
    • MAP is preferred for storing arbitrary data
    • IF / ELSE IF / ELSE code will only execute if the statement meets a condition, or is true.
    • The strict equality operator (===) checks to see if two values are equal
    • FOR LOOP
      • Uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
      • Needs another statement to interrupt the control flow
      • Contains three statements
        • 1: determines the starting point for our loop
        • 2: is the condition
        • 3: is what allows the array to iterate over each item.
        • ex: for(var x = 0; x < shapes.length; x++) {}
    • LENGTH: finds the length of an array
    • The code x++ is a shortcut way of writing x+1

Terminal + Other Tips

  • pwd: check location of the directory (print working directory)
  • ls: shows a list of files and folders
  • cd ____:move directories
  • cd .. :goes backwards
  • mkdir: make directory
  • touch ____ .txt: creates a text file
  • open ___: open files
  • code .: open VS
  • Most common way to style a webpage is by using an external CSS style sheet
    • Preferred because it allows developers to keep all their CSS rules in a separate file making design changes easier
  • Do Not Repeat Yourself (DRY)
    • Coding principle encouraging developers to reuse or share pieces of code to reduce the number of lines of code that need to be written or maintained.
    • DRY can greatly reduce the overall size and complexity of a codebase.
  • Separation of Concerns is a widely used term in web development
    • Refers to separating code into different files based on use
    • It is based on the idea that each section of code should have its own responsibility
  • REAL INTERACTIVITY on a website requires event handlers.
    • These are code structures that listen for activity in the browser, and run code in response.
  • CONTROL FLOW or the order in which a computer executes code
  • CONDITIONALS are code structures used to test if an expression returns true or not.
  • FUNCTIONS are a set of instructions that tells the computer how to perform a certain task.
  • ARRAYS can be used to store data types
  • BOOLEAN is a true or false conditional.
  • STRINGS is textual data encoded as a sequence of 16-bit unsigned integer values representing UTF-16 code units
    • Each element in the string occupies a position in the string
    • The first element is at index 0, the next at index 1, and so on.
    • Using strings to represent complex data comes with short-term benefits:
      • Easy to build complex strings with concatenation.
      • Easy to debug (what you see printed is always what is in the string)
      • Are the common denominator of a lot of APIs
    • BUT creates unnecessary maintenance burdens because it breaks LISTS when separators are used, or unexpected inputs can be put in like the escape key
  • SYMBOL is a unique and immutable primitive value and may be used as the key of an OBJECT property
  • OBJECT is a value in memory which is possibly referenced by an IDENTIFIER.
  • ARRAYS are regular objects for which there is a particular relationship between integer-keyed properties and the length property.
    • A perfect candidate to represent ordered lists
    • Typically used to hold data that is related in some way
    • Though not limited to contain only a single type of data
    • Uses a unique number called an index to pair inputs
    • Zero-indexed and sequential
  • Typed Arrays present an array-like view of an underlying binary data buffer, and are similar to arrays
    • Is an umbrella term for a range of data structures
  • These data structures take object references as keys
    • SET and WEAKSET represent a collection of unique values
    • MAP and WEAKMAP represent a collection of key-value associations.
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format, derived from JavaScript, but used by many programming languages.
