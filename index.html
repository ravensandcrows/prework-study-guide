HTML

  • The HEAD element contains information about the webpage
    • Defines three elements:
      • The charset element -> sets character encoding
      • The http-equive element sets an HTTP header for content
      • The viewport element sets the default view for our webpage
    • The HREF attribute points to the URL for the link for both BODY and HEAD
    • Can embed an entire stylesheet directly in the HTML file using a STYLE element in the HEAD element
      • The rules only apply to that page
        • can configure CSS classes and IDs -> used to identify elements, to style multiple elements in the page
    • a LINK element creates the connection to the external file. Use:
      • REL attribute specifies the relationship between the current document and the linked document resource
      • HREF links to external resource, using a relative path
  • The BODY element represents the visible content shown to the user
    • When working with visible HTML elements, the content is contained between an opening tag and a closing tag
    • DIV creates a division in the page
    • SPAN creates an inline division
    • The H1, H2, H3, H4, H5, and H6 elements represent the level of heading a given text block represents
      • H1 should only be used once per page + headers should be used in ascending order
    • The P element represents a paragraph or block of text
    • The UL, OL, and LI elements represent unordered lists, ordered lists, and list items
    • The IMG element contains information about images that are displayed on the webpage
      • The SRC attribute is the most important because it defines the location of the image
      • The ALT attribute contains a text string that describes the appearance and functionality of the image.
        • Helps with screen readers, assistive devices, and can improves SEO rankings
    • The BR element creates a line of empty space, or a line break
    • The A element (which stands for anchor) creates links to the same webpage or other webpages
  • The FOOTER appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
  • The SECTION elements are semantic elements that define a section in a document that contains thematic content
    • Creates a class =“___”; can be referenced in CSS
  • To STYLE an element directly in the HTML file use inline CSS
    • Inline styles apply to a specific HTML tag, with a style attribute from a CSS rule
    • + Quick and permanent changes
    • - less flexible

CSS

  • STYLE RULES
    • Can set multiple effects separated by commas
    • period preceding _____ (ie .card) designates CLASS selector
      • CLASS allows us to share a CSS rule to any element assigned to the CLASS
    • A SELECTOR defines the element or attributes to which the rules, or declarations, will apply; such as DIV, H1
      • DECLARATIONS contain two important components:
        • The CSS property we want to apply
        • The value of the property.
      • EX: The DECLARATION color: blue
    • Every HTML element is essentially a box that contains:
      • MARGINS
      • BORDERS
      • PADDING
      • CONTENT
    • External Style Sheet: MUST LINK to HTML
      • Link the file using a LINK element placed in the HTML's HEAD
      • Ex: LINK REL="stylesheet" HREF="./assets/style.css"
    • Sometimes, we want to apply a rule to all the elements visible on the page: use a special wild card symbol, or *
  • STYLE OPTIONS
    • COLOR uses semantic reference ie gray or by hex #
    • WIDTH, use % or px
    • MARGIN PROPERTY(how much space around the outside of an element)
      • Use px value to TOP, BOTTOM, LEFT, & RIGHT
      • AUTO can be used in place of px
      • MARGIN 0: means no space around element
    • PADDING: adds space around the content inside element
      • Increase padding = enlargen element
    • BORDER PROPERTY assigns a border with a WIDTH, HEIGHT, and COLOR
    • BOX-SHADOW adds shadow effect arond element frame:
      • uses X (+ right, - left) & Y (+ below, -above) to offset relative to element
      • BLUR and SPREAD radius, and COLOR
    • IMG
      • DISPLAY
        • BLOCK: assigns each image to start on a new lin

Terminal + Other Tips

  • pwd: check location of the directory (print working directory)
  • ls: shows a list of files and folders
  • cd ____:move directories
  • cd .. :goes backwards
  • mkdir: make directory
  • touch ____ .txt: creates a text file
  • open ___: open files
  • code .: open VS
  • Most common way to style a webpage is by using an external CSS style sheet
    • Preferred because it allows developers to keep all their CSS rules in a separate file making design changes easier
  • Do Not Repeat Yourself (DRY)
    • Coding principle encouraging developers to reuse or share pieces of code to reduce the number of lines of code that need to be written or maintained.
    • DRY can greatly reduce the overall size and complexity of a codebase.
  • Separation of Concerns is a widely used term in web development
    • Refers to separating code into different files based on use
    • It is based on the idea that each section of code should have its own responsibility
